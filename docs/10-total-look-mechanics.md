# Total look: одежда, обувь, аксессуары и причёски

Статус: дополнение к PRD и доменной логике. Этот документ описывает слой полного образа поверх базового гардероба.

## 1. Что меняется в продукте

Приложение подбирает не только одежду. Итоговый образ состоит из пяти слоёв:

1. одежда: верх, низ, платье/комбинезон, средний слой, верхняя одежда;
2. обувь;
3. аксессуары: украшения, часы, шарфы, ремни и другие небольшие детали;
4. сумки: рюкзак, crossbody, tote, mini bag, спортивная сумка;
5. hair direction: причёска, подача волос и мягкий совет по цвету;
6. makeup direction: макияж/уход как стилевой референс с возрастной политикой.

Одежда, обувь, украшения, сумки и головные уборы являются реальными предметами гардероба. Приложение не должно «добавлять» в образ вещь, которой нет у пользователя. Если подходящей вещи нет, оно может дать идею: «к этому образу подошла бы тонкая серебристая цепочка» или создать gap/wishlist item после подтверждения.

Причёска не хранится как вещь. Это styling guidance, который зависит от выбранного стиля, длины волос, текущего цвета волос и выбранной подачи.

Макияж тоже не является обязательной частью образа. Для младших возрастов приложение предлагает только уход/SPF/бальзам или не предлагает макияж вообще. Для подростков makeup direction зависит от выбранного стиля и gender/style presentation: например Stockholm — clean/minimal, emo/goth — один тёмный акцент, coquette — мягкий румянец и блеск.

## 2. Профиль внешности

В onboarding добавляется лёгкий профиль:

- gender/style presentation: feminine, masculine, neutral или not specified;
- hair length: very short/buzz, short, medium, long, very long;
- hair color: black, dark brown, brown, light brown, blonde, red, gray/ash, bright dyed, mixed, other;
- openToColorAdvice: можно ли показывать советы по оттенку.

Важно: gender/style presentation не должен запрещать причёски. Он меняет приоритет и язык рекомендаций. Например, для длинных волос и masculine presentation приложение скорее предложит low ponytail, clean flow или bun; для feminine presentation — soft waves, half-up, ribbons, face-framing layers. Neutral даёт варианты без гендерной рамки.

## 3. Как собирается образ

Recommendation engine работает в два прохода.

### A. Core outfit

Сначала собирается валидный комплект из реальных вещей:

- body base: top + bottom или one_piece;
- footwear;
- mid_layer/outerwear по погоде и событию;
- hard filters: стирка, размер, наличие вещи, dress code, ограничения родителей.

Это остаётся самым важным слоем. Красивый аксессуар не может компенсировать отсутствие обуви или недоступную вещь.

### B. Styling layer

После core outfit engine добавляет optional slots:

- jewelry/accessory;
- bag;
- headwear;
- hair suggestion.

Optional slots повышают score, если они поддерживают выбранный стиль, но не делают образ невалидным при отсутствии. Для подростков после 10 лет погодные детали вроде шапки подаются как вариант, а не как команда.

## 4. Пример: чёрные волосы + Stockholm style

Если пользователь выбирает стокгольмский стиль, имеет чёрные волосы и разрешил советы по цвету, приложение может сказать:

> Если хочется ближе к референсам «Стокгольмский», чаще подходят русый, тёмный блонд или мягкий шатен. Чёрный цвет тоже можно оставить — это направление, не правило.

Такой текст специально не звучит как оценка внешности. Он говорит не «тебе не подходит», а «в визуальных референсах этого стиля чаще встречается такой оттенок».

## 5. Роль LLM

LLM можно использовать для:

- понимания запроса: «собери стокгольмский образ, но с эмо-деталью»;
- генерации мягкого объяснения;
- локализации и перефразирования reason codes;
- разбора long-tail стилей и трендовых названий.

LLM не должен самостоятельно решать:

- какие вещи существуют в гардеробе;
- чистая ли вещь;
- подходит ли размер;
- нужно ли ребёнку обязательно менять цвет волос;
- можно ли показывать конкретный совет без разрешения пользователя/родителя.

Источник истины — structured engine и profile settings. LLM — слой языка и интерпретации.

## 6. Фото одежды и cutout

Правильный target flow:

1. Пользователь фотографирует вещь.
2. Клиент загружает исходник в object storage.
3. Vision pipeline делает segmentation/background removal.
4. В `wardrobe_items` сохраняются два поля:
   - `imageUri` — исходное/санитизированное фото;
   - `cutoutUri` — PNG/WebP с прозрачным фоном.
5. Flatlay использует `cutoutUri`. Если его ещё нет, UI временно показывает `imageUri` и статус `PENDING_CUTOUT`.

В текущем product slice реальная вырезка выполняется локальным `vision-worker` через `rembg/u2netp`: endpoint `/v1/cutout-image` принимает фото, удаляет фон, обрезает прозрачные поля и возвращает оптимизированный PNG с alpha-каналом. UI сразу использует результат в коллаже. Legacy endpoint `/v1/cutout` оставлен только для совместимости с ранним object-storage flow. Перед массовым production-запуском inline base64 нужно заменить на signed upload в object storage/CDN, добавить AV/NSFW-проверку исходников и лимиты хранения.

## 7. Текущая реализация product slice

В первом working slice уже есть:

- новые слоты гардероба: headwear, jewelry, bag;
- новые категории: belt, necklace, bracelet, ring, earrings, watch, backpack, crossbody bag, tote, hair accessory и другие;
- поля профиля: genderPresentation и hairProfile;
- генерация `hair`, `makeup` и `stylingSuggestions` внутри `OutfitOption`;
- мягкий color advice, включая пример для Stockholm style;
- отображение фото/cutout вещи в flatlay вместо чисто цветовой плитки;
- реальное удаление фона через `rembg/u2netp` с прозрачным PNG;
- quick add для украшений, сумки и головного убора;
- демо-гардероб с цепочкой, crossbody и beanie.

Дальше этот слой можно расширять через style graph: для каждого стиля хранить hair silhouettes, preferred hair colors, accessory grammar, bag shapes и forbidden clichés.
